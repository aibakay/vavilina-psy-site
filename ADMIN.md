# Панель редактирования сайта

Сайт собирается на 11ty из JSON-файлов в `src/`. Панель редактирования
(Sveltia CMS) даёт возможность менять содержимое блоков через удобные формы:
клиент правит текст → изменения сохраняются в репозиторий → Vercel
автоматически пересобирает сайт. Базы данных и отдельного сервера не нужно.

- Адрес панели: **`https://<ваш-домен>/admin/`**
- Что можно редактировать: все блоки главной, страницы продуктов, сертификаты,
  контакты, меню и SEO (см. `admin/config.yml`).
- Картинки (портреты, сертификаты) загружаются прямо из панели в папку `assets`.

## Разовая настройка (делает владелец репозитория)

Панель уже в коде. Чтобы она заработала, нужно один раз подключить вход через
GitHub. Занимает ~10 минут.

### Шаг 1. Создать GitHub OAuth App

1. Откройте https://github.com/settings/developers → **OAuth Apps** → **New OAuth App**.
2. Заполните:
   - **Application name**: `Vavilina site admin` (любое)
   - **Homepage URL**: `https://<ваш-домен>`
   - **Authorization callback URL**: `https://<ваш-домен>/api/callback`
     ⚠️ Домен должен точно совпадать с тем, на котором открывается `/admin/`.
3. Нажмите **Register application**.
4. Скопируйте **Client ID**, затем **Generate a new client secret** и скопируйте секрет.

> Если у сайта несколько доменов (например, `*.vercel.app` и свой домен),
> используйте тот, с которого будете заходить в панель. Для второго домена
> заведите отдельное OAuth App.

### Шаг 2. Добавить переменные окружения в Vercel

Проект на Vercel → **Settings → Environment Variables** → добавьте:

| Имя | Значение |
|-----|----------|
| `GITHUB_CLIENT_ID` | Client ID из шага 1 |
| `GITHUB_CLIENT_SECRET` | Client Secret из шага 1 |
| `ALLOWED_DOMAINS` | (необязательно) домен панели, например `vavilina.ru` |

После добавления нажмите **Redeploy**, чтобы переменные применились.

### Шаг 3. Дать доступ клиенту

Панель редактирует репозиторий от имени вошедшего пользователя, поэтому у него
должен быть доступ на запись:

- Репозиторий на GitHub → **Settings → Collaborators** → пригласить аккаунт
  клиента с ролью **Write**.
- Клиент принимает приглашение (придёт на почту / в уведомления GitHub).

Готово. Клиент заходит на `https://<ваш-домен>/admin/`, жмёт **Login with
GitHub**, разрешает доступ — и видит формы редактирования.

## Как пользоваться (для клиента)

1. Откройте `https://<ваш-домен>/admin/` и войдите через GitHub.
2. Слева — разделы: **Главная страница**, **Настройки сайта**, **Отдельные страницы**.
3. Выберите блок, измените поля, нажмите **Save** (сохранить).
4. Через **1–2 минуты** после сохранения сайт пересоберётся и изменения появятся
   на странице. Это не мгновенно — так и должно быть.

Полезно знать:
- Каждое сохранение — это версия в истории, любое изменение можно откатить.
- В блоке «Доверие и подход» поле «Ссылка» под карточкой можно оставить пустым.
- Во вкладках отзывов галочку «Активна по умолчанию» ставьте только у одной вкладки.
- Размеры фото (ширина/высота) меняйте только если заменили изображение на другое
  по пропорциям.

## Как это устроено (для разработчика)

| Файл | Назначение |
|------|------------|
| `admin/index.html` | Точка входа Sveltia CMS; подставляет адрес OAuth из текущего домена |
| `admin/config.yml` | Описание всех блоков и полей (коллекции) |
| `api/auth.js` | Старт OAuth: редирект на GitHub |
| `api/callback.js` | Обмен кода на токен, возврат в панель через `postMessage` |
| `.eleventy.js` | Копирует папку `admin/` в сборку `_site/` |

Контент остаётся в тех же JSON, из которых уже собирается сайт
(`src/_data/*.json`, `src/products/*/*.json`, `src/certificates/certificates.json`) —
CMS ничего не дублирует, а редактирует их напрямую.

### Возможные доработки
- Editorial Workflow (черновики + модерация через Pull Request) — если нужно, чтобы
  правки клиента проходили проверку перед публикацией.
- Создание новых страниц продуктов через панель (folder collection) — сейчас панель
  редактирует существующие; добавление новых можно включить по запросу.

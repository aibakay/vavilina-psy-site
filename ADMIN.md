# Панель редактирования сайта

Сайт собирается на 11ty из JSON-файлов в `src/`. Панель редактирования
(Sveltia CMS) даёт возможность менять содержимое блоков через удобные формы:
клиент правит текст → изменения сохраняются в репозиторий → Vercel
автоматически пересобирает сайт. Базы данных и отдельного сервера не нужно.

Вход в панель — **по паролю**. Клиенту не нужен GitHub-аккаунт: все правки
коммитятся одним общим токеном владельца.

- Адрес панели: **`https://<ваш-домен>/admin/`**
- Что можно редактировать: все блоки главной, страницы продуктов, сертификаты,
  контакты, меню и SEO (см. `admin/config.yml`).
- Картинки (портреты, сертификаты) загружаются прямо из панели в папку `assets`.

## Разовая настройка (делает владелец репозитория)

Панель уже в коде. Нужно один раз создать токен и задать пароль. ~5 минут.

### Шаг 1. Создать токен GitHub (fine-grained PAT)

1. Откройте https://github.com/settings/tokens?type=beta → **Generate new token**.
2. Настройте:
   - **Token name**: `vavilina-site-admin` (любое)
   - **Expiration**: на ваше усмотрение (можно `No expiration`; при истечении токен
     нужно будет обновить в Vercel).
   - **Repository access** → **Only select repositories** → выберите
     `aibakay/vavilina-psy-site`.
   - **Permissions** → **Repository permissions** → **Contents**: `Read and write`.
     (Остальное не трогайте — `Metadata: Read-only` добавится автоматически.)
3. **Generate token** и скопируйте значение (показывается один раз).

Токен ограничен только этим репозиторием и только контентом — ничего другого им
сделать нельзя.

### Шаг 2. Переменные окружения в Vercel

Проект на Vercel → **Settings → Environment Variables** → добавьте:

| Имя | Значение |
|-----|----------|
| `GITHUB_TOKEN` | Токен из шага 1 |
| `ADMIN_PASSWORD` | Пароль для входа в панель (придумайте сами) |

После добавления нажмите **Redeploy**, чтобы переменные применились.

### Шаг 3. Передать клиенту

Отправьте клиенту:
- ссылку `https://<ваш-домен>/admin/`
- пароль (значение `ADMIN_PASSWORD`)

Всё. Никаких аккаунтов и приглашений не требуется.

## Как пользоваться (для клиента)

1. Откройте `https://<ваш-домен>/admin/`.
2. Нажмите вход, введите пароль.
3. Слева — разделы: **Главная страница**, **Настройки сайта**, **Отдельные страницы**.
4. Выберите блок, измените поля, нажмите **Save** (сохранить).
5. Через **1–2 минуты** после сохранения сайт пересоберётся и изменения появятся
   на странице. Это не мгновенно — так и должно быть.

Полезно знать:
- Каждое сохранение — это версия в истории, любое изменение можно откатить.
- В блоке «Доверие и подход» поле «Ссылка» под карточкой можно оставить пустым.
- Во вкладках отзывов галочку «Активна по умолчанию» ставьте только у одной вкладки.
- Размеры фото (ширина/высота) меняйте только если заменили изображение на другое
  по пропорциям.

## Как это устроено (для разработчика)

| Файл | Назначение |
|------|------------|
| `admin/index.html` | Точка входа Sveltia CMS; подставляет адрес входа из текущего домена |
| `admin/config.yml` | Описание всех блоков и полей (коллекции) |
| `api/auth.js` | Вход по паролю; при верном пароле отдаёт панели общий токен |
| `.eleventy.js` | Копирует папку `admin/` в сборку `_site/` |

Контент остаётся в тех же JSON, из которых уже собирается сайт
(`src/_data/*.json`, `src/products/*/*.json`, `src/certificates/certificates.json`) —
CMS ничего не дублирует, а редактирует их напрямую. Все коммиты панели
авторства владельца токена (`aibakay`).

### Важно про безопасность
- Пароль общий, а не персональный. Журнала «кто именно менял» по именам не будет.
- После входа токен оказывается в браузере редактора (Sveltia обращается к GitHub
  API напрямую). Токен ограничен только этим репозиторием и только контентом,
  поэтому риск минимален. Если пароль скомпрометирован — смените `ADMIN_PASSWORD`
  и при желании пересоздайте `GITHUB_TOKEN`, затем Redeploy.

### Возможные доработки
- Персональные входы через GitHub (журнал правок по именам) — если позже понадобится.
- Editorial Workflow (черновики + модерация через Pull Request).
- Создание новых страниц продуктов через панель (folder collection).
